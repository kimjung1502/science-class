# 스펙트럼 관찰 실험 페이지 — 기획서 v1.6

> 대상: 통합과학1 「물질과 규칙성 — 원소의 생성과 규칙성」 스펙트럼 관찰·비교 실험
> 형식: 단일 HTML 파일 1개 · 오프라인 동작 · localStorage 자동저장 · 사진 학생 업로드
> 이 문서는 **제작 전 확정용 스펙**입니다. (아직 페이지는 만들지 않음)
> 디자인·진행 방식: 참고 사이트(유선전신기 준비 페이지) 계열 — **밝고 친근한 카드형 + 누적 스크롤**.

---

## 0. 한눈에 보기

| 항목 | 결정 |
|---|---|
| 배포 형태 | HTML 파일 1개, 인터넷·외부 라이브러리 불필요 |
| 저장 | localStorage 자동저장(새로고침 유지) + JSON 내보내기(세특용) |
| 진행 방식 | **6단계** 누적 스크롤(append). '다음'을 누르면 이전 내용은 남고 아래에 다음 단계가 펼쳐짐. 조건 충족 시 잠금 해제 |
| 잠금 단계 | ②(전부 정답), ④(**사진 업로드** + 슬릿 선택 + 분류 완료) |
| 사진 | 학생 촬영·업로드가 **기본 활동이자 ④ 통과 필수**(`REQUIRE_PHOTOS=true`, 끄면 선택). 갤러리·파일에서 선택(카메라 강제 안 함), 표시는 `data:` URL |
| 디자인 | 참고 사이트 계열 — 밝고 친근한 카드형, 스펙트럼을 포인트로, 반말 톤·이모지·핵심/더 자세히/📖교과서 태그 |

**6단계 흐름** (⑥ 우주 연결 제외)
```
① 도입·안전(목표 강조)  →  ② 복습(빈칸+힌트)  →  ③ 예상
        ↓
④ 관찰·기록  →  ⑤ 설명(변천사 아코디언+실험 애니메이션)  →  ⑥ 정리·평가(내보내기)
```

## 1. 디자인 시스템 (프론트엔드 핵심)

### 1.1 방향성 (참고 사이트 반영)
참고하신 사이트처럼 **밝고 친근한 카드형 + 누적 스크롤**로 갑니다. 학생이 위에서 아래로 읽어 내려가며, '다음'을 누르면 **이전 내용은 그대로 남고** 아래에 다음 단계가 펼쳐집니다. 주제가 '빛/스펙트럼'이므로 **스펙트럼 색 띠는 밝은 바탕 위의 포인트 요소**(진행 띠·강조 구분선·실제 스펙트럼 그래픽)로만 절제해서 씁니다.

> **시그니처**: 아래로 내려갈수록 상단 슬림 진행 띠가 보라→빨강으로 채워지고, 마지막 ⑦에서 연속 스펙트럼이 완성됨 — *스크롤로 스펙트럼을 완성해가는* 느낌(누적 스크롤과 맞물림).

**구조·톤 장치 (참고 사이트와 같은 계열)**
- **큰 번호 스테이지 1~7** + 카드형 콘텐츠.
- **`핵심` 배지** + **`더 자세히 ▾` 펼치기** + **`📖 교과서` 출처 태그**.
- **친근한 반말 톤 + 이모지 포인트**(👇 ✅ 🤔 💡)로 학생 눈높이.
- 어려운 개념(예: 에너지 준위)엔 **`더 쉬운 설명 보기 ↓`**를 아래로 펼치는 옵션(참고 사이트의 '개념 계단' 방식).

### 1.2 색 토큰 (밝은 톤 — *참고 사이트 느낌으로 제안, 정확한 색은 확정 필요*)

| `--surface-2` | `#f3eddf` | 강조 카드/입력 |
| `--line` | `#e7ddc9` | 경계선 |
| `--ink` | `#2b2b33` | 본문 |

| `--accent` | `#3b6fd6` | 인터랙션(버튼·링크) |

### 1.3 타이포그래피 (오프라인 안전 = 시스템 폰트)
- 본문/제목: `"Segoe UI", "Malgun Gothic", "맑은 고딕", system-ui, sans-serif`
- 수치/라벨(모노): `"Consolas", "D2Coding", "Courier New", monospace` — 파장(nm), 에너지 준위, `STEP 03` 같은 단계 표기에 사용
- 스케일: 목표 히어로 `clamp(28px, 5vw, 40px)` / 화면 제목 22px / 본문 16px / 캡션·모노 13px / nm 라벨 12px

### 1.4 모션·접근성 (품질 기준선)
- 전이 애니메이션은 짧고 절제. `prefers-reduced-motion` 존중(끄기).
- 키보드 포커스 링 항상 보이게(`--accent`).
- 터치 타깃 44px 이상, 색만으로 정보 전달하지 않기(아이콘·텍스트 병기).
- 반응형: PC/태블릿/폰. 폰에서는 상단 진행 띠를 얇게 + `3/7`로 축약.
- 누적 스크롤에서 새 단계가 펼쳐질 때 부드러운 스크롤 이동, `prefers-reduced-motion`이면 즉시 이동.

## 2. 공통 화면 구조 — 누적 스크롤(append)

```
┌───────────────────────────────────────────┐
│ ▓▓▓░░░░ 진행 띠(스펙트럼)  3/7  [2-9 김민수] │ ← 슬림 상단 고정
├───────────────────────────────────────────┤
│  ① 도입·안전        ✔ (그대로 남음)          │
│  ───────────────────────────────────       │
│  ② 복습             ✔ (그대로 남음)          │ ← 이전 내용은
│  ───────────────────────────────────       │    사라지지 않고
│  ③ 예상             ✔                       │    계속 쌓임
│  ───────────────────────────────────       │
│  ④ 관찰·기록   ← 지금 여기                    │
│     (콘텐츠 …)                              │
│        [ 다음으로 계속 ↓ ]                    │ ← 누르면 ⑤가
│         (잠김: "슬릿을 골라야 열려요")          │    아래에 펼쳐짐
└───────────────────────────────────────────┘
```
- **누적(append) 진행**: '다음'을 누르면 **이전 단계는 그대로 두고** 아래에 다음 단계 카드가 나타나며 **부드럽게 스크롤 이동**. 위로 스크롤하면 복습·관찰 내용을 언제든 다시 봄. → *요청하신 "복습 내용이 사라지지 않고 아래에 추가" 방식*.
- 각 단계 끝 버튼: `다음으로 계속 ↓` (잠김 단계는 비활성 + 이유 문구).
- **'이전' 버튼 없음** — 스크롤로 올라가면 됨. 상단 진행 띠의 단계 점을 누르면 해당 위치로 점프(선택 기능).
- 상단 슬림바: 스펙트럼 진행 띠 + `n/7` + 이름/반 칩 + 자동저장 표시 + `초기화`(확인 후).
- 콘텐츠 최대폭 ~720px, 카드형.

## 3. 데이터 구조 & 저장

### 3.1 localStorage 스키마 (키: `spectrum_lab_v1`)
```json
{
  "version": "v1",
  "student": { "name": "", "sid": "" },   // sid = 학번(예: 20900)
  "updatedAt": "ISO8601",
  "unlockedStep": 1,
  "review":  { "answers": {}, "solved": false, "hintsUsed": [] },
  "predict": { "sun": "미정", "bulb": "미정" },
  "observe": {
    "slit":    { "choice": "", "correct": false },
    "sun":     { "sawDarkLines": "", "classify": "흡수", "note": "" },
    "bulb":    { "classify": "", "correct": false },
    "compare": { "common": "", "diff": "" }
  },
  "explain": { "transitionsExplored": [], "modelViewed": false },
  "cosmos":  { "thinkAnswer": "" },
  "assess":  { "a1": "", "a2": "", "a3": "" }
}
```
> **사진은 본문 상태와 분리**해 각각 별도 키로 저장(예: `spectrum_lab_v1_photo_sun`). 큰 사진 하나가 실패해도 **텍스트 응답은 항상 저장**되도록 격리. 본문 상태 객체에는 사진 존재 여부/키만 기록.

- **자동저장**: 값이 바뀔 때마다 저장(디바운스). 페이지 열면 이어서 진행(resume).
- **`unlockedStep`**: 지금까지 열린 최고 단계. 새로고침하면 ①~`unlockedStep`을 모두 펼쳐 누적 스크롤 상태를 복원(학생이 ⑤까지 열었다면 ①~⑤ 모두 펼쳐진 상태로 로드).
- **모든 저장은 try/catch**로 감싸고, 실패 시 아래 §3.3 fallback 동작.
- **초기화**: 확인 절차 후 전체(본문+사진 키) 삭제.

### 3.2 내보내기(JSON)
- 버튼: `결과 내보내기(JSON)` → **본문 상태 객체 + 별도 사진 키를 합쳐 JSON으로 다운로드**.
- 파일명: `스펙트럼_{반}_{이름}_{YYYYMMDD}.json`
- 용도: 세특 근거자료(서술 응답·분류·사진 포함).

### 3.3 사진 저장 안정성 (자동 축소 + 실패 fallback)
1. **업로드 즉시 축소**: 캔버스로 가로 최대 ~1000px, JPEG ~0.7 → dataURL.
2. **저장 시도 → 실패(QuotaExceeded 등)면 단계적 재압축**: 800px/0.6 → 640px/0.5로 낮춰 재시도.
3. **그래도 실패하면 세션 유지 + 경고**: 이번 화면에는 보이되(메모리), "저장 공간이 부족해 이 사진은 새로고침하면 사라질 수 있어요. 지금 **결과 내보내기**로 백업해요." 안내.
4. **격리**: 사진 저장 실패가 텍스트 응답 저장을 막지 않음(별도 키라서).
5. 상단바에 저장 상태 표시(`저장됨` / `저장 실패 — 내보내기로 백업`).
6. **표시(미리보기)는 저장과 분리** — 고른 사진은 `FileReader`의 `data:` URL로 즉시 표시(샌드박스/CSP에서 `blob:`이 막혀도 안전). 위 축소(1~3)는 **저장 시점에만** 적용.
7. **사진 선택 방식**: 파일(갤러리) 선택. `capture` 속성 미사용 → 카메라 강제 열림 없음. `accept=".jpg,.jpeg,.png,.webp,.heic,.heif"`.

### 3.4 입력 검증 (서술형 — 공백 방지만)
- 모든 서술형(예상 이유, 공통점·차이점, 사고력, 평가 3문항)은 **`trim()` 후 비어 있으면 미완료 처리**.
- 잠금 조건도 `값.trim().length > 0`로 판정. 공백만 입력해서 다음으로 넘어가는 것 방지.
- 그 이상(최소 글자수·내용 검사)은 두지 않음 — 요청대로 **공백 방지 수준까지만**. 안내 문구: "내용을 입력해 주세요."

## 5. 화면별 상세 (와이어프레임 + 인터랙션 + 저장값 + 잠금)

### ① 도입 · 안전 — **실험 목표 강조**
```
┌──────────────────────────────────────┐
│ STEP 01                              │
│                                      │
│  ┃ 오늘의 실험 목표                    │  ← 스펙트럼 색 강조 바
│  ┃                                   │
│  간이 분광기로 여러 광원의 스펙트럼을     │  ← 히어로(가장 큼)
│  관찰·비교하고, 이를 원자의 구조와        │
│  천체의 구성 원소와 관련지어 설명한다.     │
│                                      │
│  세부 목표 1·2·3·4  (작은 목록)         │
│  ─────────────────────────────────   │
│  준비물: 간이 분광기, 스마트 기기         │
│  ⚠ 안전  ⚠ 안전  (카드)              │
│  이름[      ]  학번[ 20900 ]           │
│              [ 시작하기 → ]           │
└──────────────────────────────────────┘
```
- **히어로**: 실험 목표를 화면 최상단·최대 크기로. 스펙트럼 색 강조 바를 곁들여 열자마자 보이게.
- 세부 목표 4개(작은 목록), 준비물, 안전 카드(태양 눈 직접 관찰 금지·카메라로만 / 주위 어둡게).
- 이름·**학번**(예: `20900`, 숫자만 입력) 입력(내보내기 식별용) → `시작하기`. 상단 칩은 `학번 · 이름`.
- **저장**: `student.name/sid`
- **잠금**: 이름·학번(숫자) 입력 시 시작 활성.

### ② 스펙트럼 복습 — 빈칸 + 힌트
```
┌──────────────────────────────────────┐
│ STEP 02 · 복습                        │
│ [ (가)(나)(다) 스펙트럼 이미지 ]        │  ← 올려주신 이미지 삽입
│                                      │
│ 1. 빛을 분광기에 통과시킬 때 나타나는     │
│    여러 색의 띠를 [        ]이라 한다.   │  ← 타이핑
│                              [힌트🔒]  │
│ 2. (가)처럼 색이 이어진 것은 [   ] 스펙…  │
│ 3. (나)처럼 밝은 선은 [    ] 스펙트럼…    │
│ 4. (다)처럼 검은 선은 [    ] 스펙트럼…    │
│                                      │
│ [ 정답 확인 ]   ✔ 3/4 맞음            │
└──────────────────────────────────────┘
```
- 빈칸 타이핑, 공백·유사어 허용(예: "방출/선" 둘 다 인정).
- **힌트🔒**: 누르면 비밀번호 입력창 → `CONFIG.HINT_PW` 일치 시 힌트 공개(사용 기록 저장).
- **저장**: `review.answers`, `solved`, `hintsUsed`
- **잠금(다음)**: 4문항 전부 정답이어야 열림. (문구: "빈칸을 모두 맞혀야 다음으로 넘어가요.")

### ③ 예상

```
┌──────────────────────────────────────┐
```

### ④ 관찰 · 기록 — 핵심 (슬릿 A/B 비교·선택 + 분류)
```
┌────────────────────────────────────────────┐
│ STEP 04 · 관찰·기록                          │
│ ☀️ 안전: 태양은 분광기로 눈에 직접 대고 보지    │
│    말고, 밝은 하늘 쪽을 카메라로만 담기!         │
│ [A] 슬릿 조절(태양 촬영) — 두 사진 비교하기       │
│  ┌──────────────┐   ┌──────────────┐        │
│  │ 슬릿 넓게      │   │ 슬릿 좁게      │        │
│  │ [사진 업로드]   │   │ [사진 업로드]   │        │
│  │  (썸네일/확대)  │   │  (썸네일/확대)  │        │  ← 나란히 비교
│  │  ( ) 이걸 선택  │   │ (◉) 이걸 선택   │        │
│  └──────────────┘   └──────────────┘        │
│  Q. 이 실험에 더 적합한 사진은?  → [ 확인 ]    │
│  → 정답(좁게) 시: "슬릿이 좁아야 선이 또렷하게  │
│     분리돼 태양의 검은 선(흡수선)까지 보입니다." │
│ ────────────────────────────────────────    │
│ [B] 광원 관찰·분류                            │
│  ┌──────────────┐   ┌──────────────┐        │
│  │ 백열등 [업로드] │   │ 태양 [업로드]   │        │  ← 두 스펙트럼도
│  │ 종류:(연속…)   │   │ ☀️ 눈 직접 X    │        │     나란히 비교
│  │               │   │ 검은 선 찾음?   │        │

│  └──────────────┘   └──────────────┘        │
│   → 백열등=연속(확정) / 태양=원리상 흡수(정오답X)│
```

- **안전 문구 반복**: ①에만 두지 않고, **태양을 촬영하는 [A]·[B] 상단에 `☀️ 눈으로 직접 보지 말고 카메라로만` 콜아웃을 다시 노출**(관찰 직전 상기).
- **[A] 슬릿 비교·선택 (핵심 추가)**: 태양을 넓게/좁게 **두 장 업로드 → 좌우로 나란히 비교**(썸네일, **탭하면 확대**해서 선 또렷함 직접 대조). 각 사진 아래 라디오로 **"이 실험에 더 적합한 사진은?" 선택 → 확인**.
  - 정답 = **좁게**. 질문은 "또렷한 쪽"이 아니라 **"이 실험에 더 적합한"**으로 두어 *목적 적합성*을 판단하게 함. 정답 시 이유(또렷한 분리 → 흡수선까지 관찰) 피드백.
  - 미업로드/미선택 시 안내 문구.
- **[B] 광원 비교·분류**: 백열등·태양 사진을 **같은 나란히 비교 뷰**로 배치해 두 스펙트럼을 눈으로 대조하며 분류·서술.
  - **백열등 = 연속(확정 채점)**.
  - **태양**: "검은 선(프라운호퍼선) 찾았나?" 예/아니오 → **예상→관찰→설명**(원리상 흡수, 못 봤으면 슬릿·기기 한계 안내, 정오답 X).
  - 나란히 보이는 두 사진을 근거로 **공통점·차이점** 서술.
- **재사용 컴포넌트**: `PhotoCompare(좌, 우, {선택가능})` — [A]는 선택형(적합한 것 고르기), [B]는 비교·서술형으로 같은 뷰를 공유.
- **사진 필수(구현)**: `CONFIG.REQUIRE_PHOTOS=true`. 사진을 올리지 않으면 슬릿 정답을 골라도 통과 불가.
  - 슬릿 '확인'은 **넓게·좁게 사진 2장이 있어야** 유효(없으면 "먼저 사진을 올려줘").
  - [B]의 백열등·태양 사진도 통과 요건(끄고 싶으면 플래그 `false`).
  - 선택은 파일(갤러리), 표시는 `data:` URL(FileReader) — §3.3-6·7 참조.
- **저장**: `observe.slit.choice/correct`, `observe.bulb/sun/compare`, 사진 별도 키.
- **잠금(다음)**: **사진 업로드**(REQUIRE_PHOTOS 시) + 슬릿 **적합한 것 정답 선택** + 백열등 분류(연속) + 태양 관찰 응답 + 공통/차이 **입력(공백 제외, trim)**.

### ⑤ 설명 — 원자 모형 변천사(아코디언) + 실험 애니메이션
```
┌────────────────────────────────────────────┐
│ STEP 05 · 설명                              │
│ [원자 모형 변천사 — 세로 타임라인·아코디언]    │
│  ▸ 돌턴 1803                                │
│  ▸ 톰슨 1897  ── 펼치면 ──┐                  │
│  ▸ 러더퍼드 1911          │ 등장 배경         │
│  ▾ 보어 1913 (기본 펼침)   │ 실험 내용+애니메이션│
│  ▸ 오비탈 현대            │ 알 수 있는 것(타이핑)│
│                          └ 💡힌트🔒(비계)     │
└────────────────────────────────────────────┘
```
- **순서**: 변천사(돌턴→톰슨→러더퍼드→**보어**→오비탈)를 먼저 제시하고, **보어**에서 에너지 준위·전자 전이로 '선'을 설명한 뒤 **오비탈**로 마무리.
- **각 모형 = 아코디언**. 펼치면 **등장 배경(ㅇㅇㅇ 실험) · 실험 내용 · 알 수 있는 것(학생 타이핑)** 구조. (예시 답안 없음 — 학생이 스스로 추론)
- **실험 애니메이션(관찰용)**:
  - **톰슨(음극선)**: 직진 / 자석(휨) / 물체(그림자) / 바람개비(회전) / 전기장((+)극으로 휨) 모드. 방전관 속 빛나는 입자 흐름.
  - **러더퍼드(알파 산란)**: `톰슨 예측`(모두 통과) ↔ `실제 결과`(대부분 통과 + 일부 큰 산란 + 되튐) 비교.
  - **보어**: 에너지 준위 다이어그램 + 전이 클릭 → 스펙트럼 선 점등, ↓방출/↑흡수 토글(같은 위치 확인). 수소 발머선 n3→2 656 / 4→2 486 / 5→2 434 / 6→2 410 nm.
- **결론 비노출 + 잠긴 힌트(비계)**: 각 실험은 **결론을 직접 보여주지 않음**. `💡 힌트 🔒`(비밀번호 `HINT_PW`) → **답을 말하지 않고 추론을 유도하는 단서**만 제공. 학생은 관찰 → "알 수 있는 것"에 스스로 추론 타이핑.
- **오비탈**: 등장 배경(보어 모형의 한계 + 전자의 **파동성**) + **영의 이중슬릿 실험(애니메이션)** — 빛·전자가 두 틈을 지나 스크린에 간섭무늬(파동의 증거) → 위치를 한 점으로 못 정하고 '**확률 분포=오비탈**'로 추론. `💡힌트🔒` 비계. *(점밀도·경계면·4가지 양자수 등 상세는 교육과정 제외로 삭제)*
- **저장**: `explain.knows.{dalton…orbital}`(타이핑), `transitionsExplored`.
- **잠금**: 보어 전이 1개 이상 탐색(느슨). 타이핑은 저장하되 통과 필수는 아님(옵션으로 전환 가능).

## 8. 콘텐츠·정답키 (초안 — 확정 필요)

### 8.1 실험 목표(①)
- **목표**: 간이 분광기로 여러 광원의 스펙트럼을 관찰·비교하고, 이를 원자의 구조와 천체의 구성 원소와 관련지어 설명할 수 있다.
- **세부목표**
  1. 연속·방출·흡수 스펙트럼을 구분한다.
  2. 태양·백열등의 스펙트럼을 관찰하고 종류를 분류한다.
  3. 스펙트럼이 나타나는 까닭을 전자의 에너지 준위로 설명한다.
  4. 별(태양)의 흡수 스펙트럼으로 구성 원소를 아는 원리를 설명한다.

### 8.2 복습 빈칸 정답(②)
| # | 정답 | 허용 표기 |
|---|---|---|
| 1 | 스펙트럼 | spectrum |

## 10. 배포 · 공유 (GitHub Pages)

### 10.1 구성
- 파일명 **`index.html`** 로 저장소 루트에 배치(자산·백엔드 불필요, 이미지도 base64로 내장).
- **Settings → Pages → Source: Deploy from a branch → `main` / `/ (root)`** → 몇 분 뒤 `https://<아이디>.github.io/<저장소>/` 로 접속.
- GitHub Pages는 HTTPS라 사진 업로드·localStorage 정상 동작.

### 10.2 데이터·프라이버시
- 정적 호스팅 = 서버 없음. **학생 사진·응답은 각자 브라우저에만 저장**, GitHub에는 페이지 코드만 공개됨.
- 취합은 학생이 ⑦에서 내보낸 **JSON을 제출**받는 방식(중앙 자동수집 아님).

### 10.3 공개 저장소 주의
- 소스가 공개되므로 **`CONFIG.HINT_PW`·정답키가 노출**됨(형성평가용이라 대개 무방).
- 신경 쓰이면: 비공개 저장소+비공개 Pages(GitHub Pro) / 또는 정답 하드체크를 자기점검형으로 완화.

### 10.4 업데이트·캐시
- 파일 수정 시 캐시로 옛 버전이 보일 수 있음 → 상단에 **버전 표기(예: `v1`)** + "안 바뀌면 새로고침(Ctrl+F5)" 안내.

## 11. 구현 현황 — `스펙트럼_실험_페이지_①-⑤.html`

### 11.1 진행 상황
- **①~⑤ 구현 완료**(누적 스크롤로 이어짐), **⑥ 정리·평가 미구현**(다음 회차, 스텁). **총 6단계**(⑥ 우주 연결 삭제).
- 현재 빌드는 **인메모리**(새로고침 시 초기화). 전 과정 **저장·복원(§3)·JSON 내보내기(§3.2)**는 마지막 조립 회차에 부착.
- 초기 파일(`…미리보기.html` ①~②, `…①-④.html`)은 이 파일로 대체됨.

### 11.2 구현되어 확정된 UI/동작
- **상단 진행 띠**(스펙트럼, **6단계 기준** 채워짐, 시그니처) + `STEP 0n · 라벨` + `학번·이름` 칩.
- **①** 히어로 목표+세부목표+준비물, 앰버 안전 카드, **이름·학번(20900, 숫자)** 입력 시 `시작하기` 활성.
- **②** `핵심`·`📖 교과서` 배지 → (가)(나)(다) 이미지 → 빈칸 4개 타이핑 → 문항별 `힌트🔒`(비번 `1234`) → `더 자세히 ▾` → `정답 확인` `✔/✗`. 전부 정답이라야 다음.
- **③** 태양·백열등 예상(연속/방출/흡수) + 이유(선택). 둘 다 고르면 다음.
- **④** 사진 **파일 선택**·`data:` 표시·탭 확대 → 슬릿 넓게/좁게 비교 + '더 적합한 것' 선택 채점(사진 2장 필수) → 백열등=연속·태양 검은선 예/아니오·공통/차이 서술. **사진 미업로드 시 통과 불가**(`REQUIRE_PHOTOS`).
- **⑤** 원자 모형 변천사 **아코디언**(돌턴~오비탈, 보어 기본 펼침). 각 아코디언 = 등장 배경·실험 내용·**알 수 있는 것(타이핑)**. **실험 애니메이션**: 톰슨(음극선 5모드)·러더퍼드(예측↔실제)·보어(에너지 준위·전이+방출/흡수 토글)·**오비탈(영의 이중슬릿 간섭무늬)**. **결론 비노출 + `💡힌트🔒`(비번 `1234`) 비계**로 추론 유도. 보어 전이 1회 클릭 시 다음.
- **⑥ 정리·평가**(스텁): 서술 3문항 + JSON 내보내기(다음 회차).
- **누적 스크롤**: 버튼 누르면 이전 내용 남고 다음 단계가 아래에 열림.
